# Why does Expo exist? What problems does it solve?
Expo serves as a powerful platform to create mobile apps. It is built on top of React Native (which is cross-platform) and provides lots of useful features to speed up and help developers to create and maintain mobile applications usiing Javascript / Typescript, and is open source. It provides a wide range of useful quality additions - from various Expo components that can be used in React Native code, to EAS (Expo Application Services), that provide a way to build, deploy and monitor apps, as well as managing updates and workflows. Expo can manage projects of various size - from a small, simple app projects to enterprice grade platforms.

Expo helps abstracting many lower level APIs and provides ready-made components for building code, as well as infrastructure for managing the app outside of development, which makes React Native way more accessible to developers who primarily use Javascript. It has created the whole ecosystem of services and development packages, that help developers to focus more on user experience. It has integrated into React Native ecosystem very well, to the point where React Native team suggests using Expo when building apps with React Native. From official React Native documentation: "<...> To build a new app with React Native, we recommend a framework like Expo."

### Some features that are provided by Expo:
* Expo Go - acts as a sandbox, provides a way to experiment with new features. This app can be installed on Android and IOS from official play stores.
* Native app modules - provide modules for managing camera, audio, location, etc.
* OTA (Over The Air) updates - enables a way to push updates for live users without them needing to update the app through the play stores.
